# Writing Voice

A TypeScript toolchain for revising Markdown into a configured writing voice.

The system has two layers:

- Vale handles objective, inspectable voice linting.
- The host agent handles rewriting with a strict voice contract generated by the CLI or MCP server.

## Quick Start

```bash
pnpm install
pnpm build
pnpm wetzler prepare README.md --audience "technical collaborator" --goal "explain the tool"
```

## Validation

CI runs on every push and pull request. These are the same local checks behind the visible badges:

| Signal                  | Command                      | What it proves                                                            |
| ----------------------- | ---------------------------- | ------------------------------------------------------------------------- |
| Build                   | `pnpm build`                 | Compiles the core package, MCP server, and CLI.                           |
| Tests                   | `pnpm test`                  | Runs the Vitest behavior suite across all packages.                       |
| Type-check              | `pnpm typecheck`             | Builds first, then checks strict TypeScript contracts.                    |
| MCPB runtime            | `pnpm validate:mcpb`         | Builds the MCPB bundle, validates it, and smoke-tests MCP initialization. |
| Codex plugin runtime    | `pnpm validate:codex-plugin` | Validates the Codex plugin manifest and smoke-tests the launcher.         |
| Production dependencies | `pnpm audit --prod`          | Audits shipped dependency paths for known vulnerabilities.                |

## Main Commands

```bash
wetzler lint README.md --json
wetzler lint --stdin --path draft.md --json
wetzler lint /path/to/draft.md --repo-root /path/to/wetzler
wetzler prepare draft.md --audience "engineering manager" --goal "fit the configured voice"
wetzler samples add samples/post.md --label "blog post" --weight 2
wetzler learn propose --samples "voice/samples/*.md"
wetzler learn validate <proposal-id>
wetzler learn accept <proposal-id>
wetzler mcp
```

`lint` and `prepare` exit non-zero when Vale has a runtime error, such as a missing `.vale.ini`.
Lint findings are still returned as normal results so agents can revise against them.

## Repository Layout

- `voice/profile.yml` is the durable voice contract.
- `samples/` stores example source samples you can import or inspect.
- `voice/samples/` stores writing samples and sample metadata.
- `voice/proposals/` stores review-gated voice update proposals.
- `styles/voice/` stores Vale rules.
- `styles/config/vocabularies/voice/` stores accepted and rejected terms.
- `packages/core/` exposes the public TypeScript API.
- `packages/cli/` exposes the `wetzler` command.
- `packages/mcp-server/` exposes the MCP server used by plugins.
- `plugins/` contains Codex and Claude Code wrappers.
- `extensions/claude-desktop/` contains the MCPB manifest scaffold.
- `.agents/plugins/marketplace.json` exposes the Codex plugin as a repo-local marketplace entry.

## Rewriting Contract

The CLI and MCP server do not call a model directly. They return a revision packet with:

- the original Markdown,
- Vale findings,
- voice rules,
- structural preservation rules,
- and output instructions for the host agent.

By default, the agent should return revised Markdown without writing files unless explicitly asked.

## Learning Updates

`wetzler learn propose` creates a review-gated proposal under `voice/proposals/`.
The proposal includes deterministic observations from the selected samples and candidate profile
changes. Edit the proposal before accepting if the suggested change does not match the evidence.

`wetzler learn validate <proposal-id>` checks a proposal in a temporary Vale repo without applying
it, and `wetzler learn accept <proposal-id>` runs the same validation before applying. The MCP server
can create and validate proposals, but applying proposals is intentionally CLI-only so a human remains
in the loop for changes to the durable voice contract.

## Distribution

For Codex, use the repo-local marketplace at `.agents/plugins/marketplace.json`.

For Claude Desktop or other MCPB-aware clients:

```bash
pnpm validate:mcpb
pnpm pack:mcpb
```

The MCPB package contains the native Vale binary from the machine that built it, so generated bundles
are platform-specific. Build one package per target platform and architecture.
